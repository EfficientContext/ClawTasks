# claw-tasks

Prefill-heavy, multi-turn document analysis tasks for benchmarking LLM inference optimizations on [OpenClaw](https://openclaw.ai) agent workloads.

## Overview

18 enterprise documents (260 KB), 30 tasks across 4 categories, 122 turns total. Documents share ~70% template content — a natural stress test for context optimization.

| Category | Tasks | Description |
|----------|-------|-------------|
| [`claw-tasks/commercial/`](claw-tasks/commercial/) | 5 | Contract values, SLA targets, payment terms, renewal timelines, cost analysis |
| [`claw-tasks/legal/`](claw-tasks/legal/) | 7 | Liability, IP ownership, termination, dispute resolution, warranties, confidentiality, force majeure |
| [`claw-tasks/compliance/`](claw-tasks/compliance/) | 11 | Data protection, insurance, subcontracting, BCP/DR, personnel, acceptance testing, audit rights, open source, cross-border, breach response |
| [`claw-tasks/strategic/`](claw-tasks/strategic/) | 7 | Change management, vendor comparison, knowledge transfer, security assessment, exit strategy, board summary, negotiation prep |

## Data

A technology company (TechVentures Pte Ltd) manages four vendor relationships. The workspace contains:

| Documents | Files | Size |
|-----------|-------|------|
| Service Agreements | 4 | 182 KB |
| Amendments | 4 | 19 KB |
| Master Service Agreement | 1 | 9 KB |
| Vendor Assessments | 4 | 16 KB |
| Internal Policies | 2 | 13 KB |
| Board Minutes | 1 | 8 KB |
| NDAs | 2 | 12 KB |

Contracts share a common legal template (Articles 1-16, Schedules A-B) with vendor-specific terms. Amendments reference originals. Assessments quote SLA targets. Policies are cross-referenced in the MSA.

## Quick Start

```bash
# Run all tasks
python scripts/run_bench.py --gpu 0

# Run one category
python scripts/run_bench.py --category commercial

# Run specific tasks
python scripts/run_bench.py --scenarios s01_commercial_terms s02_liability_review

# Analyze results
python scripts/analyze.py results/results.jsonl
```

## Workload Characteristics

| Property | Value |
|----------|-------|
| Avg input tokens (turn 0) | ~25K |
| Avg input tokens (turn 3) | ~65K |
| Max input tokens | ~93K |
| Avg output tokens | ~740 |
| Input/output ratio | ~60:1 |
| Content overlap between contracts | ~70% |
